AI RELIABILITY CASE STUDY

AuraGateway stopped when the evidence stopped.

A read-only account of two provider lineages that did not produce trustworthy numeric cache evidence—and the harness controls that prevented unsupported performance claims.

THE ANSWER

The A/B/C cache benchmark was not eligible to run.

Groq completed inference but omitted the required cached-token field. OpenRouter returned HTTP 401 before successful Hy3 inference. Neither result supports a cache hit, miss, saving, or latency conclusion.

PROVIDER LINEAGES

Different failures. Same fail-closed decision.

CLAIM CONTROL

What the evidence allows—and what it blocks.

METHOD

The experiment narrowed before it executed.

  1. Freeze the question. Separate deterministic context from route affinity.
  2. Validate the adapter. Preserve absent, null, zero, and positive states.
  3. Bound the calls. One-time authorization, append-only evidence, no rerun.
  4. Run only when eligible. Stop when the measurement channel fails.
  5. Publish the negative result. Retain non-claims as first-class evidence.
EVIDENCE BOUNDARY

This is a replay surface, not a live benchmark.
